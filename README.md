# Sand & Bone — Генератор сражений (v4)

## Что изменилось в этой версии

1. **Армии вынесены из кода в `armies.json`** — `index.html` больше не содержит данные армий внутри `<script>`, он загружает их через `fetch('armies.json')` при старте. Базовые армии (Кузнецы Хашута, Рабы Тьмы) лежат там же, где и раньше — это просто другой файл рядом.
2. **Новые армии добавляются вместе с Claude в чате, не через браузер.** Раньше планировался импорт PDF прямо в приложении через Claude API — отказались от этого: это требовало от тебя своего платного API-ключа и усложняло пайплайн без особой выгоды. Вместо этого: ты присылаешь мне PDF официального варскролла в чате, я читаю его, формирую правильный JSON по нужной схеме и сразу добавляю в `armies.json` — ты можешь сразу посмотреть результат и попросить поправить. Это надёжнее (я вижу весь текст PDF целиком, а не урезанный кусок) и без какого-либо API-ключа вообще.
3. **Новые армии получают свой акцентный цвет автоматически** — цвет определяется детерминированно по хэшу ключа армии из палитры в 8 оттенков и применяется через CSS-переменную `--army-accent`. Когда я добавляю новую фракцию в JSON, ничего вручную в стилях прописывать не нужно — цвет появится сам.
4. **Кнопка «⬇ Выгрузить armies.json»** в разделе «Моя армия» — просто скачивает текущее состояние файла армий, который сейчас загружен в приложении. Это локальный бэкап на всякий случай, без какой-либо отправки куда-либо.

## Файлы в этом пакете

- `index.html` — основной файл приложения
- `armies.json` — данные армий (сейчас: Кузнецы Хашута, Рабы Тьмы)

Оба файла должны лежать **в одной папке** рядом друг с другом — `index.html` подгружает `armies.json` относительным путём.

## Как добавить новую армию

1. Прислать мне в чате PDF официального Spearhead-варскролла на английском (как ты уже делал для Hashut/Slaves).
2. Я читаю PDF, формирую новую запись для `armies.json` в том же формате, что и существующие армии (боевая черта, способность отряда, улучшения генерала, юниты с оружием и способностями) — с пересказом текста своими словами на русском, не дословным переводом.
3. Присылаю тебе обновлённый `armies.json` — можно сразу посмотреть, попросить что-то поправить.
4. Когда всё устраивает — заливаешь обновлённый `armies.json` на GitHub (см. инструкцию ниже), и армия становится доступна сразу обоим игрокам.

Никакого API-ключа, никакого pdf.js в браузере, никакого риска, что что-то распознается неточно без твоей проверки — весь процесс идёт здесь, в чате, под контролем.

## Как опубликовать на GitHub Pages (пошагово)

### Если репозитория ещё нет

1. Зайти на [github.com](https://github.com), нажать **New repository** (зелёная кнопка справа сверху или `+` → `New repository`).
2. Указать имя, например `sand-and-bone`. Видимость — **Public** (для бесплатного GitHub Pages на личном аккаунте репозиторий должен быть публичным, либо нужен платный план для приватных Pages).
3. Можно сразу поставить галочку **Add a README file** — неважно, ниже всё равно загрузим свои файлы.
4. Нажать **Create repository**.

### Загрузка файлов

5. На странице репозитория нажать **Add file → Upload files**.
6. Перетащить туда `index.html` и `armies.json` (оба файла одним разом, в корень репозитория — не в подпапку).
7. Внизу страницы — поле commit message, можно оставить как есть или написать что-то вроде "v4: армии в отдельном json". Нажать **Commit changes**.

### Включение GitHub Pages

8. В репозитории открыть вкладку **Settings** (вверху).
9. В левом меню — **Pages**.
10. В разделе **Build and deployment → Source** выбрать **Deploy from a branch**.
11. В **Branch** выбрать `main` (или `master`, смотря как называется основная ветка) и папку `/ (root)`. Нажать **Save**.
12. Подождать 1-2 минуты — GitHub соберёт страницу. Адрес появится в этом же разделе Pages, обычно вида:
    `https://<твой-логин>.github.io/<имя-репозитория>/`

### Обновление файлов в будущем (например, когда добавляем новую армию)

13. На странице репозитория открыть `armies.json`, нажать значок карандаша (**Edit**), вставить новое содержимое (которое я тебе пришлю), **Commit changes**.
14. Либо: **Add file → Upload files** и просто перезалить файл с тем же именем — GitHub предложит заменить существующий.
15. Сайт на GitHub Pages обновляется автоматически в течение ~1 минуты после коммита — отдельного действия не требуется.

## Известные ограничения

- Открытие `index.html` двойным кликом (`file://`) не даст подгрузить `armies.json` — браузеры блокируют `fetch` для локальных файлов по соображениям безопасности. Для локального теста нужен локальный сервер (например, `python3 -m http.server` в папке с файлами) или просто открывать через сам GitHub Pages. Приложение покажет всплывающее предупреждение, если `armies.json` не подгрузился.
