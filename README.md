# Sand & Bone — Генератор сражений (v4)

## Что изменилось в этой версии

1. **Армии вынесены из кода в `armies.json`** — `index.html` больше не содержит данные армий внутри `<script>`, он загружает их через `fetch('armies.json')` при старте. Базовые армии (Кузнецы Хашута, Рабы Тьмы) лежат там же, где и раньше — это просто другой файл рядом.
2. **Импорт новых армий из PDF через Claude API** — в разделе «III. Моя армия» появилась панель «Добавить армию из PDF»:
   - пользователь вставляет свой Claude API-ключ (хранится только в `localStorage` браузера, уходит напрямую на `api.anthropic.com`, никуда больше);
   - загружает официальный PDF Spearhead-варскролла на английском;
   - текст PDF извлекается в браузере через `pdf.js`;
   - весь текст одним запросом уходит в Claude (`claude-sonnet-4-6`) с инструкцией вернуть JSON по той же схеме, что используется для Кузнецов Хашута / Рабов Тьмы;
   - результат сразу сохраняется в `localStorage` и появляется в списке армий — без шага ручной проверки/правки (так и попросили).
3. **Импортированные армии получают свой акцентный цвет автоматически** — цвет определяется детерминированно по хэшу ключа армии из палитры в 8 оттенков и применяется через CSS-переменную `--army-accent`, а не через статичный CSS-класс. Не нужно ничего прописывать руками в стилях для новой фракции.
4. **Кнопка «⬇ JSON»** у каждой импортированной армии — выгружает её как `<key>.json`, чтобы вручную добавить в общий `armies.json` и закоммитить в репозиторий. После этого армия станет видна обоим игрокам навсегда (а не только в браузере того, кто её импортировал).
5. **Кнопка «Удалить эту импортированную армию»** — чистит конкретную армию из `localStorage` браузера.

## Важно: PDF должен быть текстовым, не сканом

`pdf.js` вытаскивает только текстовый слой PDF. Если варскролл — это скан-картинка без текстового слоя, извлечение не сработает (приложение покажет ошибку об этом). Официальные PDF от Games Workshop обычно текстовые, так что для них всё ок.

## Важно: ограничение по объёму

PDF обрабатывается одним вызовом API целиком (весь извлечённый текст уходит в одном запросе, до ~60000 символов). Для одного варскролла-армии Spearhead (обычно несколько страниц) этого с большим запасом достаточно. Если когда-нибудь понадобится грузить что-то намного большее (полный батлтом, например) — это уже другая задача, текущая реализация на неё не рассчитана.

## Файлы в этом пакете

- `index.html` — основной файл приложения
- `armies.json` — данные базовых армий (Кузнецы Хашута, Рабы Тьмы)

Оба файла должны лежать **в одной папке** рядом друг с другом — `index.html` подгружает `armies.json` относительным путём.

## Как опубликовать на GitHub Pages (пошагово)

### Если репозитория ещё нет

1. Зайти на [github.com](https://github.com), нажать **New repository** (зелёная кнопка справа сверху или `+` → `New repository`).
2. Указать имя, например `sand-and-bone`. Видимость — **Public** (для бесплатного GitHub Pages на личном аккаунте репозиторий должен быть публичным, либо нужен платный план для приватных Pages).
3. Можно сразу поставить галочку **Add a README file** — неважно, ниже всё равно загрузим свои файлы.
4. Нажать **Create repository**.

### Загрузка файлов

5. На странице репозитория нажать **Add file → Upload files**.
6. Перетащить туда `index.html` и `armies.json` (оба файла одним разом, в корень репозитория — не в подпапку).
7. Внизу страницы — поле commit message, можно оставить как есть или написать что-то вроде "v4: импорт армий из PDF". Нажать **Commit changes**.

### Включение GitHub Pages

8. В репозитории открыть вкладку **Settings** (вверху).
9. В левом меню — **Pages**.
10. В разделе **Build and deployment → Source** выбрать **Deploy from a branch**.
11. В **Branch** выбрать `main` (или `master`, смотря как называется основная ветка) и папку `/ (root)`. Нажать **Save**.
12. Подождать 1-2 минуты — GitHub соберёт страницу. Адрес появится в этом же разделе Pages, обычно вида:
    `https://<твой-логин>.github.io/<имя-репозитория>/`

### Обновление файла в будущем

13. Когда нужно обновить `index.html` или `armies.json` (например, вручную добавить в `armies.json` импортированную армию) — на странице репозитория открыть нужный файл, нажать значок карандаша (**Edit**), вставить новое содержимое, **Commit changes**.
14. Либо: **Add file → Upload files** и просто перезалить файл с тем же именем — GitHub предложит заменить существующий.
15. Сайт на GitHub Pages обновляется автоматически в течение ~1 минуты после коммита — обновление действия не требует.

### Добавление импортированной армии в общий файл навсегда

16. Кто-то из игроков импортирует армию через PDF в браузере → у него локально появляется новая армия + кнопка «⬇ JSON» рядом с ней.
17. Скачать этот JSON-файл (например, `tzeentch_arcanites.json`).
18. Открыть `armies.json` в репозитории на GitHub (Edit), вручную вставить содержимое скачанного файла как новый ключ внутрь объекта верхнего уровня — то есть по образцу `"hashut": {...}, "slaves": {...}, "tzeentch_arcanites": {...вставленное содержимое...}`.
19. Commit changes — через минуту армия появится у всех, кто откроет страницу (даже без своего PDF/API-ключа).

## Известные ограничения

- Открытие `index.html` двойным кликом (`file://`) не даст подгрузить `armies.json` — браузеры блокируют `fetch` для локальных файлов по соображениям безопасности. Для локального теста нужен локальный сервер (например, `python3 -m http.server` в папке с файлами) или просто открывать через сам GitHub Pages.
- Качество импорта через PDF зависит от Claude и от того, насколько чисто извлекается текст — иногда таблицы оружия в PDF визуально сложные, и распознавание может промахнуться в деталях. Раз ручной проверки нет, стоит при первом использовании новой импортированной армии сверить пару способностей с оригинальным PDF.
- API-ключ хранится в `localStorage` браузера в открытом виде — это нормально для личного использования на своём устройстве, но не стоит так делать на общем/чужом компьютере.
